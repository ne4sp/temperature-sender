# temperature-sender

В этом репозитории два подпроекта:

- `esp32/` — прошивка для ESP32, которая отправляет температуру на сервер по HTTP `POST`.
- `server/` — веб‑сервер, который принимает температуру и отображает её на странице с графиком.

## 1) Сервер (Node.js)

### Запуск

Установите зависимости и запустите:

```bash
cd server
npm install
npm run dev
```

По умолчанию сервер слушает `http://0.0.0.0:8080`.

- Веб‑страница: `http://<IP_ПК>:8080/`
- Приём температуры: `POST http://<IP_ПК>:8080/api/temperature`
- История: `GET http://<IP_ПК>:8080/api/history`
- Погода (улица): `GET http://<IP_ПК>:8080/api/weather` (см. ниже)

### Хранение истории на диске

История датчиков сохраняется в файл **`server/data/history.json`** (папка `server/data/` в `.gitignore`, при `git pull` не затирается).

- Переменная **`DATA_DIR`** — если нужно хранить файл вне репозитория (например на сервере: `DATA_DIR=/var/lib/tsender`).

### Погода на улице (Open‑Meteo, без API‑ключа)

На сервере задайте координаты:

- **`WEATHER_LAT`** — широта (например `55.7558`)
- **`WEATHER_LON`** — долгота (например `37.6176`)

Опционально: **`WEATHER_TTL_MS`** — как долго кэшировать ответ (по умолчанию 10 минут).

### Защита от мусорных данных (рекомендуется)

Можно включить простой API‑ключ:

- На сервере задайте переменную окружения `API_KEY` (любой длинный секрет).
- На клиенте (ESP32/скрипт) добавляйте заголовок `x-api-key: <API_KEY>`.

Также есть простое ограничение частоты запросов (по IP):

- `RATE_LIMIT_PER_MIN` (по умолчанию 120 запросов/мин).

### Формат данных (JSON)

`POST /api/temperature`

```json
{
  "celsius": 23.5,
  "humidity": 50.0,
  "deviceId": "esp32-kitchen",
  "ts": 1710000000000
}
```

- `celsius` — обязательное число
- `humidity` — опционально (влажность в %)
- `deviceId` — опционально (строка)
- `ts` — опционально (миллисекунды Unix). Если не передано — сервер поставит своё.

## 2) ESP32 (Arduino IDE / PlatformIO)

Файл: `esp32/temperature_sender/temperature_sender.ino`

Нужно указать:

- `WIFI_SSID`, `WIFI_PASS`
- `SERVER_URL` (пример: `http://192.168.0.10:8080/api/temperature`)

Скетч умеет отправлять температуру (по умолчанию “фейковую” синусоиду для теста). Если вы подключите датчик (например, DS18B20), замените функцию чтения температуры.

## 3) ESP32 (MicroPython) + DHT11

См. `esp32-micropython/` — пример для ESP32 на MicroPython с датчиком DHT11, который отправляет **температуру и влажность** на `POST /api/temperature`.
